# 要件
- データ分析コンペティションの目的を明確にする
本コンペティションの課題では、中小企業向け融資データをもとに、各企業が債務を返済できるかどうか（デフォルトするかどうか）を予測するタスクに取り組んでいただきます。
参加者の皆さまには、企業の基本情報や融資条件（借入金額、金利タイプ、期間、業種、企業年齢など）をもとに、最終的なローンの返済ステータス（返済完了 or デフォルト）を予測するモデルを構築していただきます。

- 現状の進行状況をAIが把握できるように　進捗状況を記述
例：　データの概要を把握し、前処理の方針を検討中

- 使用するデータセットの説明を含める
パス、ファイル名、データの形式など
データの内容は　MUFG_competition/prompt/description.txt を参照する
データのパス：
MUFG_competition/data/raw/sample_submit.csv
MUFG_competition/data/raw/train.csv
MUFG_competition/data/raw/test.csv

-　データの出力場所を明記する
データの出力場所：
前処理済みデータは以下の場所に保存されます。
MUFG_competition/data/processed/
出力するデータは以下の場所に保存されます。
MUFG_competition/data/output/

- 評価方法は F1スコアを使用することを明記する
- 評価用データセットの説明を含める
コンペ最終日までの評価（暫定評価）は評価用データセットの一部で評価し、コンペ終了後の評価（最終評価）は評価用データセットの残りの部分で評価します。
暫定評価は評価用データセット全体の50%、最終評価は評価用データセット全体の50%となります。
リーダーボードはコンテスト終了時に自動的に最終評価に切り替わり、それを元に最終順位を決定します。このため、開催中と終了後では順位が大きく変動する場合もあります。

- 使用するのはJupyter Notebookでデータの分析を行い、Pythonファイルでモデルの実装を行うことを明記する
notebookの場所:
MUFG_competition/notebooks/
pythonファイルの場所:
MUFG_competition/src


- Jupyter Notebookのセルには、コードの説明をコメントとして記載することを明記する

- Pythonファイルには、関数の定義とその説明をコメントとして記載することを明記する

- データ分析の結果をAIが解釈するためのファイルを作成することを明記する
AIが解釈するためのファイルの場所:
MUFG_competition/prompt/interpretation.txt
- データ分析の結果をAIが解釈するためのファイルには、以下の内容を含めることを明記する
  - データの概要
  - データの前処理方法
  - モデルの選定理由
  - モデルの評価方法
  - 結果の解釈
